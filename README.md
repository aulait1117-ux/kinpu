# 菌譜（きんぷ / Kinpu）

臨床検査技師のための、細菌検査Wiki。スマホだけで完結する。

**https://aulait1117-ux.github.io/kinpu/**

ブラウザで開いて「ホーム画面に追加」すれば、アプリのように使える。サーバーは要らない（AI整理を使うときだけ、その本文がCloudflare Workerを一瞬通る）。

---

## 何ができるか

### 菌データベース（86菌）

菌一覧はカード表示。**グラム陽性＝暖色、グラム陰性＝寒色**で、一覧から瞬時に見分けられる。

菌を1つ開くと、その画面に必要なものが全部そろっている。

| | |
|---|---|
| ⚠️ 報告時の注意 | 現場で間違えやすいところ。最上部に赤で出る |
| 🧪 感受性を入れる | 薬の S / I / R を押すと、**次にやるべき確認試験・報告のしかた・届出義務**が出る |
| 💊 効く薬 / 効かない薬 | 自然耐性（効かないはず）と、必ず感性のはずの薬（耐性ならおかしい） |
| 🧬 耐性機序 | ESBL・AmpC 等へのリンク |
| 📖 基本情報 | 学名・グラム染色像・培養特徴・主な感染症・感染対策・学習メモ |
| 📝 自分のノート | その菌について書いたことが集まる |

収録：グラム陽性球菌12／グラム陽性桿菌5／腸内細菌目16／ブドウ糖非発酵菌5／グラム陰性その他11／**嫌気性菌23**／抗酸菌4／真菌9／その他1。足りなければ画面から自分で追加できる。

### 抗菌薬データベース（25系統・66薬剤）

「イミペネムって何系だっけ」を1秒で引くための表。系統ごとに作用機序とスペクトラム。系統がわかれば、自然耐性の意味も芋づるで思い出せる。

### 耐性機序データベース（14機序）

ESBL / AmpC / KPC / NDM / IMP / OXA / MRSA / VRE / CRE / MDRP / CRAB / MLSb / BLNAR / VISA。

各ページに、概要・原因遺伝子・壊される薬と残る薬・検査時の注意・落とし穴を置いた。

### 相互リンク

**菌 ⇔ 抗菌薬 ⇔ 耐性機序 ⇔ ノート** が双方向にリンクする。ESBLのページから大腸菌へ飛べるし、大腸菌のページからESBLへ戻れる。

### ノート（AI整理つき）

走り書きでいい。写真を貼って「✨ AIで整理する」を押すと、こう変わる。

```
入力：血培陽性だったのでグラム染色して先生に電話した
      GPCクラスターだったのでブドウ球菌っぽいと伝えた
      先輩に「コアグラーゼの結果が出るまで断定するな」と言われた
```

↓

📌 要点 ／ ⚠️ 注意点 ／ 🛠 作業手順 ／ 💡 現場のコツ ／ 📞 報告・連絡 ／ ✨ 一言まとめ ／ タグ ／ 関連する菌・抗菌薬・耐性機序

ノートには**種類**を付ける。🦠細菌検査の知識／🛠作業手順・SOP／🚨機器トラブル／📋症例／🔬機器の操作／📞申し送り・報告／💡先輩メモ。

SOP・トラブル・症例・機器マニュアルを別々のデータベースにはしていない。すべて「種類の違うノート」として1つの仕組みで扱う。だから検索もタグもAI整理も相互リンクも、全部に同じように効く。新しい分類は `app.js` の `KINDS` に1行足すだけで増える。

### 検索

ヘッダーの 🔍 か、ホームの検索バー。菌・抗菌薬・耐性機序・ノート・タグを横断する。PCでは Ctrl+K でも開く（スマホでは押せないので、あくまでおまけ）。

## データはどこにあるか

**この端末の中だけ**（localStorage と IndexedDB）。外のサーバーには一切送らない。写真もアップロードしない。

裏返すと、ブラウザの履歴を消したり機種変すると消える。⚙️設定 → 「書き出す（JSON）」で、ときどき自分にメールしておく。

例外は「✨AIで整理」を押したときだけ。そのときは本文と写真が整理用のWorkerを通る。Workerは何も保存しない。

## 患者情報を入れない

保存時に、カルテID・氏名・生年月日・病棟番号らしき文字列を機械で検出して警告する。ただし完全ではない。**写真に検体ラベルや氏名が写り込んでいないか、撮った直後に自分で確認する。**

## 判定ルールは「下書き」である

`rules.js`・`mechanisms.js`・`details.js` に入っている中身は、すべて `verified: false` で出荷している。画面には「未確認 — 鵜呑みにしない」と表示される。

CLSI M100 の最新版と自施設のSOPで1件ずつ突き合わせ、確認できたものだけ「確認済み」にチェックを付ける。**ブレークポイントの数値は載せていない。** 年次改訂で変わるので、載せると古くなった瞬間に嘘になる。

医学的な中身は3ファイルに集約してある。読んで直せる。

## 構成

| ファイル | 中身 |
|---|---|
| `index.html` | 画面とCSS（かわいい・Notion風・ダークモード対応） |
| `app.js` | 画面遷移・検索・判定・保存・AI連携 |
| `rules.js` | 菌86＋薬剤パネル＋自然耐性＋必ず感性＋確認試験ルール30 |
| `drugs.js` | 抗菌薬25系統・作用機序・スペクトラム |
| `mechanisms.js` | 耐性機序14。菌と薬をつなぐハブ |
| `details.js` | 菌の詳細（グラム染色像・培養・感染症・感染対策・報告注意） |
| `kinpu-ai-worker/` | AI整理のCloudflare Worker。APIキーはシークレットに置く（コードには書かない） |

### AI整理Workerの運用

```bash
cd kinpu-ai-worker
npx wrangler secret put ANTHROPIC_API_KEY   # キーを入れる（チャットやコードに貼らない）
npx wrangler deploy
```

1日50回で頭打ちにしてある（KVでカウント）。URLが漏れても請求が伸び続けない。上限は `wrangler.jsonc` の `DAILY_LIMIT` で変えられる。

### 品質チェック

Playwrightで、86菌＋14機序の全ページ描画・ルールの発火・データ整合性（ルールが参照する薬剤が菌のパネルに載っているか等）を機械検査している。「画面は動くのにルールが永久に発火しない」類の欠陥は目視では見つからないため。
